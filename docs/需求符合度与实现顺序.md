# 需求符合度与实现顺序

> 评审基准：`docs/软件工程与计算II-课程设计需求分析.md`、`docs/api-design.md`  
> 评审对象：当前工作区代码  
> 评审日期：2026-06-23  
> 更新日期：2026-06-23（补充遗漏项，优化阶段顺序）

## 一、总体结论

当前项目属于"数据库和后端 API 骨架基本形成，但业务闭环、前端和 AI 功能尚未完成"的阶段，尚不满足课程设计验收要求，也暂不具备完整演示条件。

- 后端可以编译并在当前基础设施环境中启动。
- 数据库、实体、Repository 和 Controller 覆盖了大部分设计模块。
- 多个核心接口只有路径和数据结构，实际业务仍为 TODO、模拟结果或空操作。
- 前端只有一个根路由标题页面，全部用户操作界面尚未实现。
- AI 智能批阅、RAG 答疑和语义相似度分析均未真正调用 Spring AI。
- 文件上传、下载和在线预览未接入 MinIO。
- 后端测试用例为 0；前端构建与 lint 均失败。

## 三、关键阻断问题

### P0：安全和权限

1. `/api/v1/auth/me` 和个人资料更新接口直接返回 `User` 实体，`password` 没有禁止 JSON 序列化（`User.java:33`）。
2. 提交列表和提交详情没有教师、课程成员或提交者权限校验，任意登录用户可能枚举其他用户的提交。
3. AI 会话删除没有校验会话所有者，`ChatController.java:54` 历史查询调用了错误的方法（返回会话列表而非历史消息）。
4. 匿名话题和匿名回复仍返回 `authorId`（`TopicService.java:53,77`），匿名功能实际上失效。
5. 学生可以调用创建课程和备课区接口，系统没有按照全局用户角色限制功能（`CourseService.java:40`）。
6. `TopicReply` 回复时 `parentId` 不校验是否属于当前话题（`TopicService.java:88`），可构造跨话题楼中楼。

### P0：主流程不可用

7. `SubmissionService.getAssignmentCourseId()` 固定返回 `0L`（`SubmissionService.java:132`），导致教师评分和退回流程无法通过权限检查。
8. `FileService` 没有实际上传、下载和预览实现（`FileService.java` 全部 stub），作业附件、提交文件和课程资料链路全部中断。
9. 作业 Controller 丢弃截止时间和附件 ID（`AssignmentController.java:54,57` 硬编码 `null`）。
10. 备课草稿发布不会创建目标业务对象（`LessonDraftService.java:60-67`），却会删除草稿，数据永久丢失。
11. `SubmissionService.submit()` 删除旧文件后未关联新文件（`SubmissionService.java:66-68`），重复提交后提交记录无文件。
12. `CourseService.joinByCode()` 教师加入固定为 `STUDENT`（`CourseService.java:79`），多人共管不可用。

### P0：数据完整性

13. `CourseMember`、`TopicReply`、`Notification` 三个实体有 `deleted` 字段但缺少 `@SQLRestriction("deleted = 0")`——JPA 查询会将已删除的记录连同活跃记录一起返回（`CourseMember.java:27`, `TopicReply.java:24`, `Notification.java:24`）。

### P1：构建与交付

14. `npm run build` 因 TypeScript project reference 配置错误而失败。
15. `npm run lint` 因缺少 ESLint 9 配置而失败。
16. Docker Compose 只包含基础设施，不能一键启动完整系统。
17. 后端虽然编译成功，但没有任何自动化测试。
18. Spring Boot 4.0.7 下 `ObjectMapper` 不是自动配置 Bean——`JwtUtil` 和 `SecurityConfig` 已临时修复，但需在工程基线阶段建立规范解法。

## 四、推荐实现顺序

实现顺序以"先形成安全、可测试的主链路，再增加附加和 AI 能力"为原则。
其中阶段 1b（最小文件能力）是阶段 2 主链路的前置依赖，不能跳过。

### 阶段 1a：修复工程基线和安全问题

目标：项目可以稳定构建，接口具备基本输入校验和权限边界。

1. **修复前端构建链**：修复 `tsconfig.node.json` project reference 问题使 `npm run build` 通过；补充 ESLint 9 配置使 `npm run lint` 通过。
2. **修复启动依赖**：为 `JwtUtil` 和 `SecurityConfig` 的 `ObjectMapper` 建立规范注入方式（显式 `@Bean` 或自建实例）。
3. **保护敏感字段**：在 `User.password` 上添加 `@JsonIgnore`（`User.java:33`），审计所有返回实体的 Controller。
4. **修复 P0 数据完整性**：为 `CourseMember`（`CourseMember.java:27`）、`TopicReply`（`TopicReply.java:24`）、`Notification`（`Notification.java:24`）添加 `@SQLRestriction("deleted = 0")`；为 `CourseMember` 和 `Submission` 补充 `@Table(uniqueConstraints=...)`。
5. **修复密码泄露**：`/api/v1/auth/me` 改为返回 DTO 而非 `User` 实体。
6. **补齐课程角色权限**：`createCourse()` 限教师；`joinByCode()` 根据全局角色分配课程角色。
7. **修复提交越权**：`SubmissionController` 所有方法添加课程成员或提交者校验。
8. **修复聊天越权**：`ChatController.getHistory` 修正为调用 `getHistory()`；`deleteSession` 添加所有者校验。
9. **修复匿名数据泄露**：匿名话题和回复的响应中屏蔽 `authorId`。
10. **修复跨话题伪造**：`TopicService.reply()` 添加 `parent.topicId == topicId` 校验。
11. **修复发布数据丢失**：`LessonDraftService.publish()` 改为先创建目标实体成功后再删除草稿。

验收标准：前后端构建通过；密码不出现在任何响应中；核心越权测试全部通过。

### 阶段 1b：最小文件能力（主链路前置依赖）

目标：使阶段 2 的作业附件和提交文件不再是 TODO。

> ⚠️ 阶段 2 主链路涉及"附件上传"和"文件提交"，两者都依赖真实文件能力。
> 如果跳过此阶段直接做阶段 2，附件和文件部分将保持 TODO，主链路不完整。

1. 设计临时文件实体（`TempFile`：id, url, size, name, uploaderId, expireAt, associated）。
2. 实现 MinIO 上传（大小 ≤ 50MB、扩展名白名单、MIME 校验）。
3. 实现 MinIO 预签名下载 URL（30 分钟有效期）和 PDF/图片预览 URL。
4. 实现"先上传后关联"：临时文件 → 业务实体关联。
5. 定时清理 24 小时内未关联的孤儿文件（`@Scheduled`）。

验收标准：可上传文件并获取真实 URL；可下载；PDF/图片可预览；过期文件自动清理。

### 阶段 2：打通基础业务闭环

目标：能够演示"注册登录 → 创建/加入课程 → 发布作业 → 提交 → 批阅 → 查看结果"。

1. 完成课程列表 DTO（含课程名、封面、成员数、角色），归档/恢复/退课/删除语义及课程卡片排序。
2. **前端并行**：登录页、课程列表页、课程详情页。
3. 修复作业 Controller：正确解析 `deadline` 和 `attachmentIds`（`AssignmentController.java:54,57`）。
4. 作业状态可见性：学生只能查询 `PUBLISHED` 作业；已发布作业只能修改 `deadline` 和 `content`。
5. 重构 `SubmissionService`：注入 `AssignmentRepository`，正确校验课程、身份、状态和重复提交规则；修复文件关联逻辑（`SubmissionService.java:66-68`）。
6. 实现评分、退回、催交并发送通知。
7. **前端并行**：作业详情页、提交页、批阅页。
8. 为上述主链路增加 Service 集成测试和 API 测试。

验收标准：教师和学生可以分别完成全部基础流程，且不同角色不能访问不属于自己的数据。

### 阶段 3：完成附加功能

目标：达到"良"等级所需的资料、话题和备课能力。

1. 完善资料目录移动、递归删除和跨课程归属校验。
2. 完善话题匿名响应、讨论开关、回复关系校验和通知。
3. 限制备课区为教师专属。
4. 实现 `LessonDraftService.publish()`：根据草稿类型反序列化 `contentJson` 调用对应的 `AssignmentService.create()` / `MaterialService.create()` / `TopicService.create()`，成功后删除草稿。
5. **前端并行**：资料管理页、话题讨论页、备课区页、通知中心页。

验收标准：三个附加模块都能独立完成创建、查询、修改、删除和发布操作。

### 阶段 4：实现 AI 智能批阅

目标：形成第一个真实可演示的 AI 功能。

1. 抽取纯文本、Word 和 PDF 作业内容。
2. 校验 Rubric 权重、满分和 Prompt 模板。
3. 使用 Spring AI `ChatClient` 和结构化输出生成预评分、评语和建议。
4. 对 AI 输出进行范围校验、异常处理和结果持久化。
5. 学生提交后异步触发，教师可重新触发、修改和确认发布。
6. 实现批量任务状态追踪，而不是只返回待处理数量。
7. 增加 Mock 模型测试和至少一组受控真实模型验收用例。

验收标准：相同输入可稳定生成符合 JSON Schema 的结果，失败不会影响正常提交，教师拥有最终决定权。

### 阶段 5：实现课程 RAG 答疑

目标：基于每门课程独立知识库回答并给出来源。
> 可与阶段 6（相似度）共享 Embedding 基础设施。

1. 建立资料、作业、答案、话题和公告的索引事件。
2. 实现文档解析、分块、Embedding 和 Qdrant 写入/删除。
3. 检索时强制使用 `courseId` 过滤，避免跨课程数据泄露。
4. 拼接检索上下文和受限历史消息，调用 LLM 生成回答。
5. 返回引用名称、片段和来源 ID。
6. 修复 `ChatController.getHistory`（当前调用的是 `listSessions`）；修复会话列表、历史分页和所有者权限。
7. **前端并行**：AI 答疑页面和引用跳转。

验收标准：回答引用真实课程内容，不同课程之间无法相互检索数据，多轮追问可用。

### 阶段 6：实现语义相似度分析

目标：生成可解释的作业相似度报告。
> 可与阶段 5 共享 Embedding 基础设施，两个模块可并行开发。

1. 提取并标准化提交文本。
2. 批量生成 Embedding 并计算余弦相似度矩阵。
3. 按阈值持久化相似提交对并标记疑似抄袭。
4. 实现相似段落定位和高亮。
5. 增加历史作业批次作为可选比较范围。
6. **前端并行**：相似度报告列表、详情和可视化。
7. 使用人工构造的相同、改写和无关文本建立回归测试。

验收标准：报告包含提交对、分数、阈值判断和可解释高亮，教师可查看且学生不可访问。

### 阶段 7：部署、文档和最终验收

目标：满足课程提交和答辩要求。

1. 为后端、前端和 Nginx 增加 Dockerfile。
2. 更新 Compose，使前端、后端、MySQL、Redis、MinIO 和 Qdrant 可一键启动。
3. 增加健康检查、依赖顺序、环境变量模板和持久化卷。
4. 更新 README，保证端口和环境变量名称与代码一致。
5. 编写测试报告、部署说明和 AI 辅助编程使用报告。
6. 标注 AI 生成代码，整理关键 Prompt 和人工审查记录。
7. 执行教师、学生、权限、异常、文件、AI 和部署全流程验收。

验收标准：全新环境只需配置必要密钥并执行一次 Compose 命令即可启动，验收用例和提交文档齐全。

## 六、优先级摘要

- **P0：** 密码泄露（`User.password` 无 `@JsonIgnore`）、提交越权、聊天越权（`getHistory` 返回错误数据）、匿名失效（authorId 泄露）、跨话题伪造（parentId 无校验）、评分主流程（`getAssignmentCourseId` 恒返回 `0L`）、真实文件服务（全部 stub）、教师无法多人共管（`joinByCode` 固定 STUDENT）、**Entity 软删除注解缺失**（`CourseMember`/`TopicReply`/`Notification`）、草稿发布数据丢失。
- **P1：** 前端构建、基础业务页面、通知、测试、完整 Docker 部署。
- **P2：** 资料、话题和备课区完整性。
- **P3：** AI 批阅、RAG 答疑、相似度分析及对应评测。
