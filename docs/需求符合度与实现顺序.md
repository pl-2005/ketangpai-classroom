## 一、总体结论

当前项目处于"后端核心链路基本闭环、P0 安全问题已清零、前端仅有登录注册页面"的阶段。经过 6/25 两轮集中修复（MinIO 文件服务 + P0 安全与数据问题），原先标记的 13 项 P0 阻断问题已全部解决，项目从"不可演示"推进到"后端主链路可走通（需配合 Postman/curl 测试）"。

- 后端 14 个 Controller、114 个 Java 源文件，可编译并在基础设施环境中启动。
- **P0 阻断问题已清零**：3 个 Entity `@SQLRestriction` 补齐、评分权限链路修复、文件服务从 stub → 完整 MinIO 实现、作业 deadline/attachmentIds 解析、草稿发布不再丢失数据、聊天接口修复、全部已知越权和数据泄露路径封堵。
- 课程管理、作业发布、作业提交、在线批阅、文件管理、话题讨论、备课区、AI 答疑模块的核心接口均已完成权限校验和业务逻辑。
- 前端仅有登录和注册两个页面；课程、作业、资料、话题、备课、通知、AI 等全部业务页面待实现。
- AI 智能批阅、RAG 答疑和语义相似度分析的核心逻辑仍依赖 Spring AI 集成（实体与接口骨架已就绪）。
- 通知发送（作业发布、催交、评分、退回等场景）尚未与业务服务集成。
- 后端测试 0 个；前端 `npm run build`/`npm run lint` 状态待验证。

## 六、推荐实现顺序

### 阶段 1a：修复工程基线 ✅ 已完成

> 前后端构建链、`ObjectMapper` Bean、敏感字段保护、P0 数据完整性、权限校验 — 全部完成。

### 阶段 1b：最小文件能力 ✅ 已完成

> MinIO 上传/下载/预览/清理、TempFile 实体、定时清理调度、bucket 初始化 — 全部完成。

### 阶段 2：打通基础业务闭环（当前应开始）

目标：能够通过前端页面演示"注册登录 → 创建/加入课程 → 发布作业 → 提交 → 批阅 → 查看结果"。

1. **前端业务页面（优先）**：课程列表页、课程详情页、首页路由守卫、作业详情页、提交页、批阅页。
2. 完善作业状态可见性：学生只能查询 `PUBLISHED` 作业；已发布作业只能修改 `deadline` 和 `content`。
3. 完善 `SubmissionService.submit()` 的文件关联：TempFile → SubmissionFile 记录创建。
4. 实现通知集成：在作业发布/催交/评分/退回时调用 `NotificationService` 创建通知。
5. 为上述主链路增加 Service 集成测试和 API 测试。

验收标准：教师和学生可以通过前端页面完成全部基础流程。

### 阶段 3：完成附加功能

目标：达到"良"等级所需的资料、话题和备课能力。

1. 完善资料目录移动、递归删除和跨课程归属校验。
2. 完善话题匿名响应（前端隐藏作者信息）、讨论开关独立接口。
3. 完善备课区 publish 后的前端页面。
4. **前端并行**：资料管理页、话题讨论页、备课区页、通知中心页。

### 阶段 4：实现 AI 智能批阅

1. 抽取纯文本、Word 和 PDF 作业内容。
2. 校验 Rubric 权重、满分和 Prompt 模板。
3. 使用 Spring AI `ChatClient` 和结构化输出生成预评分、评语和建议。
4. 学生提交后异步触发，教师可重新触发、修改和确认发布。
5. 实现批量任务状态追踪（异步执行 + 进度查询）。

### 阶段 5：实现课程 RAG 答疑

1. 建立资料、作业、答案、话题的索引事件。
2. 实现文档解析、分块、Embedding 和 Qdrant 写入/删除。
3. 检索时强制使用 `courseId` 过滤。
4. 返回引用名称、片段和来源 ID。

### 阶段 6：实现语义相似度分析

1. 提取并标准化提交文本。
2. 批量生成 Embedding 并计算余弦相似度矩阵。
3. 按阈值持久化相似提交对并标记疑似抄袭。
4. 实现相似段落定位和高亮。

### 阶段 7：部署、文档和最终验收

1. 为后端、前端和 Nginx 增加 Dockerfile。
2. 更新 Compose 一键启动全部服务。
3. 编写测试报告、部署说明和 AI 辅助编程使用报告。
4. 标注 AI 生成代码，整理关键 Prompt 和人工审查记录。
5. 全流程验收。
