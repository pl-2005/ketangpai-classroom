# 软件工程与计算II — 课程设计需求分析

> 基于《软件工程与计算II课程设计题目(学生必读)-2026版（AI辅助版）》整理  
> 分析日期：2026-06-22  
> 目标等级：**优（满分）**  
> 团队规模：**3 人**

---

## 一、项目概述

开发一个 **Web 端智能课堂管理系统**（参考「课堂派」），支持教师和学生两套身份，覆盖课程管理、作业批阅、资料管理、话题讨论、备课区，以及 AI 智能批阅、AI 答疑机器人、AI 作业相似度分析三大 AI 创新功能。

---

## 二、技术选型（已确认）

| 层级 | 技术 | 说明 |
|------|------|------|
| **前端** | React + Ant Design / shadcn/ui | SPA 单页应用 |
| **后端** | Spring Boot 3.x | RESTful API |
| **AI 集成** | Spring AI | 统一大模型调用（智能批阅、答疑机器人、相似度分析） |
| **数据库** | MySQL 8.x | 业务数据持久化 |
| **文件存储** | MinIO | 作业附件、课程资源、头像等文件（S3 兼容、自托管、Docker 友好） |
| **缓存** | Redis | Session、通知消息队列、热点数据缓存 |
| **向量存储** | Qdrant Cloud | RAG 知识库的文档 Embedding 存储，使用已有云集群 |
| **部署** | Docker Compose | 一键启动全部服务（前端 Nginx + 后端 + MySQL + MinIO + Redis） |
| **版本控制** | GitHub | 团队协作，分支管理 |

### 为什么选 MinIO？

- 一行 `docker run` 即可部署，零配置
- 完全兼容 S3 API，Spring Boot 有 `io.minio:minio` 官方 SDK
- 自带 Web 管理界面，方便调试
- 不需要接入任何云服务，本地/服务器均可

## 四、功能模块

### 4.1 基础功能（必做，对标评分标准"及格/中"）

#### 账号系统

- **注册**：选择身份（教师/学生），填写用户名、密码、邮箱等
- **登录**：账号密码登录，JWT 鉴权

#### 教师端

| 模块 | 功能点 |
|------|--------|
| **课堂主页** | 列出所有课程；创建新课程；加入其他教师的课程（多人共管）；编辑/删除课程；归档（"仅归档自己"或"归档全部"）；归档管理（恢复/永久删除）；课程卡片拖拽排序 |
| **课程详情页** | 查看课程成员列表；发布作业（文字描述 + 附件上传）；发布后所有学生收到通知 |
| **作业详情页** | 查看学生提交列表；**催交**（学生收到通知）；**在线预览批阅** Word/PDF；压缩包需下载后批阅；评分并返回（学生收到通知） |

#### 学生端

| 模块 | 功能点 |
|------|--------|
| **课堂主页** | 列出已加入课程；通过课程号加入课程；归档/退课；归档管理（恢复/退课）；课程卡片拖拽排序 |
| **课程详情页** | 查看课程成员列表；展示作业列表 |
| **作业提交页** | 多文件上传；版本管理（覆盖旧版本） |

### 4.2 附加功能（必做，对标评分标准"良"）

| 模块 | 功能点 |
|------|--------|
| **资料管理** | **教师**：上传附件资源 + 添加外链资源；创建文件夹层级结构；移动/删除/下载资料。<br>**学生**：浏览目录树，下载资料 |
| **话题讨论** | 师生均可发起话题（类似贴吧帖子）；支持匿名回复；教师可置顶、编辑、删除话题；教师可开关讨论功能；学生仅可编辑/删除自己的话题 |
| **备课区**（教师专属） | 提前准备作业草稿、上传资源、起草话题；发布时从备课区一键导入到课程；草稿箱管理 |
| **AI 作业评价** | 教师为作业配置提示词 + 评分标准；学生提交后 AI 自动给出评语和改进建议；学生可据此修改作业后重新提交 |

### 4.3 AI 创新功能（必做，对标评分标准"优"）

以下 3 项 AI 功能全部基于 **Spring AI** 实现，统一管理大模型调用。

#### 功能一：AI 智能批阅助手

> 核心提效功能，也是答辩亮点

- 教师在发布作业时设定 **评分标准**（Rubric）：维度、权重、期望答案要点
- 学生提交后，AI 自动对作业进行 **预评分 + 生成评语**
- 教师进入批阅页可看到 AI 预评分结果，**确认或修改**后一键发布
- 支持对不同文件类型（Word/PDF/纯文本）进行批阅
- 可配置 Prompt 模板，教师自定义批阅风格（严厉/鼓励/详细/简洁）

**技术要点**：Spring AI ChatClient + 结构化输出（JSON Schema 限定评分格式）

---

#### 功能二：AI 课程答疑机器人

> 学生端使用频率最高的 AI 功能

- 每个课程拥有独立的 AI 答疑助手
- 基于 **RAG（检索增强生成）** 架构：
  - 索引来源：课程资料、作业题目与答案、话题讨论、公告
  - 学生提问 → 向量检索相关课程内容 → 拼接上下文 → LLM 回答
- 回答时附带 **引用来源**（"此答案参考自第三章课件"）
- 支持对话历史，多轮追问

**技术要点**：Spring AI + Embedding 模型 + Qdrant Cloud 向量存储，文档分割与索引

---

#### 功能三：AI 作业相似度分析

> 将"查重"升级为 AI 语义查重

- 对全班提交的作业进行**语义级别相似度计算**（非简单字符串匹配）
- 使用文本 Embedding 将每份作业向量化 → 余弦相似度矩阵
- 教师端展示相似度报告：谁和谁相似、相似段落高亮
- 设定阈值，超阈值自动标记为"疑似抄袭"
- 支持跨作业批次对比（本次提交 vs 历史提交）

**技术要点**：Spring AI EmbeddingClient + 向量相似度计算

## 五、非功能需求

| 类别 | 要求 |
|------|------|
| **界面** | 美观、操作流畅，参考课堂派交互，适配桌面端 |
| **安全** | JWT 鉴权、密码 BCrypt 加密、文件访问权限控制、SQL 注入防护 |
| **可用性** | Docker Compose 一键启动，无需手动配置数据库连接等 |
| **AI 规范** | AI 生成的代码需注释标注，关键 Prompt 可配置化，提交《AI 辅助编程使用报告》 |
